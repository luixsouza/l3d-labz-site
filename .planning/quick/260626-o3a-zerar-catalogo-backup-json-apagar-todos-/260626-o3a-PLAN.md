<task type="auto">
  <name>Task 1: Comando limpar_catalogo (backup JSON + delete seguro com guard)</name>
  <files>apps/catalog/management/commands/limpar_catalogo.py</files>
  <action>
Criar novo management command no estilo dos existentes (espelhar `podar_galeria.py`: docstring de módulo em pt-br com bloco "Uso:", `BaseCommand`, `add_arguments`, `handle` com `self.stdout.write`/`self.style`).

Comportamento:
- Docstring de módulo (pt-br) explicando o PORQUÊ: apagar Product é seguro porque `OrderItem.product` é SET_NULL (pedidos mantêm snapshot product_name/unit_price), a galeria cai por CASCADE (desejado), e Category é PROTECT então NÃO se apaga categorias. Por isso há backup obrigatório antes (reversível) e o guard `--confirmar`.
- `add_arguments`: `--confirmar` (`action="store_true"`) — sem ela o comando roda como dry-run. Opcionalmente também aceitar `--dry-run` explícito (no-op, já é o default) só por clareza/ergonomia, mas o estado padrão (nenhuma flag) JÁ é dry-run.
- `handle`:
  1. Contar `Product.objects.count()` e `ProductImage.objects.count()` ANTES e imprimir ("Produtos: N · Imagens de galeria: M").
  2. Se NÃO `--confirmar`: imprimir aviso "[DRY-RUN] Nenhum dado foi apagado. Rode com --confirmar para executar." com a contagem do que SERIA apagado e retornar (não apaga, não faz backup).
  3. Se `--confirmar`:
     a. Gerar timestamp com `django.utils.timezone.now().strftime("%Y%m%d-%H%M%S")` (NÃO usar Date.now / datetime.now ingênuo).
     b. Montar caminho de backup sob `settings.BASE_DIR / "backups"` (criar o diretório com `Path.mkdir(parents=True, exist_ok=True)` se não existir). Arquivo: `catalogo-<timestamp>.json`.
     c. Chamar `django.core.management.call_command("dumpdata", "catalog.Product", "catalog.ProductImage", indent=2, output=str(caminho))`. Imprimir o caminho do backup gerado e o tamanho/contagem.
     d. Apagar: `Product.objects.all().delete()` (galeria cai por cascade; OrderItem.product vira NULL). Manter Category intacta — NÃO tocar.
     e. Imprimir contagem DEPOIS (`Product.objects.count()` deve ser 0; `Category.objects.count()` inalterado) com `self.style.SUCCESS`.
- NÃO incluir limpeza de media órfã neste comando (risco de apagar o que não deve) — documentar na docstring que arquivos físicos em media/products/ não são removidos por este comando, intencionalmente.
- `help` da classe em pt-br resumindo o comportamento e citando o default dry-run.

Imports: `from pathlib import Path`, `from django.conf import settings`, `from django.core.management import call_command`, `from django.core.management.base import BaseCommand`, `from django.utils import timezone`, `from apps.catalog.models import Product, ProductImage` (e Category só se for contar).
  </action>

`python manage.py limpar_catalogo` (sem flag) imprime a contagem e o aviso de dry-run sem apagar nada. `python manage.py limpar_catalogo --confirmar` gera `backups/catalogo-<timestamp>.json` e deixa `Product.objects.count() == 0`, com `Category.objects.count()` inalterado.
